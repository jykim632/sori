## Sori 라이선스 전략

### 이중 라이선스 구조

```
┌─────────────────────────────────────────────────────────┐
│  위젯 (@sori/core, @sori/react)                          │
│  → MIT 라이선스 (완전 오픈)                               │
│  → 목적: 널리 퍼지게, 설치 허들 낮추기                     │
│  → 누구나 자유롭게 사용, 수정, 배포 가능                   │
└─────────────────────────────────────────────────────────┘
                      │
                      ▼ API 호출
┌─────────────────────────────────────────────────────────┐
│  백엔드 + Admin (@sori/web)                              │
│  → AGPL-3.0 또는 BSL 라이선스                            │
│  → 셀프호스팅 가능하지만, 상업적 SaaS는 제한               │
│  → 경쟁사가 그대로 가져다 SaaS 만드는 것 방지              │
└─────────────────────────────────────────────────────────┘
```

## 라이선스 비교

| 라이선스 | 특징 | 장점 | 단점 |
|----------|------|------|------|
| **MIT** | 완전 자유 | 채택률 높음 | 보호 없음 |
| **Apache 2.0** | MIT + 특허 보호 | 기업 친화적 | 보호 약함 |
| **AGPL** | 수정 시 소스 공개 (SaaS 포함) | 강력한 보호 | 일부 기업 기피 |
| **BSL** | N년 후 오픈소스 전환 | 유연함 | 복잡함 |

## 추천: AGPL 적용 시 효과

### 시나리오 1: 경쟁사가 코드 가져다 SaaS 운영
```
경쟁사: Sori 백엔드 코드 복사 → 자체 SaaS 런칭
     ↓
AGPL 의무: 수정된 소스 코드 전체 공개 필수
     ↓
결과: 우리와 동일한 코드로 경쟁해야 함 → 차별화 어려움
```

### 시나리오 2: 기업이 셀프호스팅
```
기업: Sori 백엔드를 자사 서버에 설치 (내부 사용)
     ↓
AGPL: 외부에 서비스 제공하지 않으면 공개 의무 없음
     ↓
결과: 내부 사용은 자유롭게 가능
```

### 시나리오 3: 기업이 수정 후 외부 서비스
```
기업: Sori 수정 → 자사 고객에게 피드백 기능 제공
     ↓
AGPL: 수정된 소스 코드 공개 필요
     ↓
결과: 커머셜 라이선스 구매 유도 가능
```

## 수익 모델

### 1. 호스팅 SaaS (주력)
- 무료 플랜: 1 프로젝트, 월 100개 피드백
- Pro 플랜: 무제한 프로젝트, 월 10,000개 피드백
- Team 플랜: 팀원 관리, 분석 기능

### 2. 커머셜 라이선스 (부가)
- AGPL 제약 없이 사용하고 싶은 기업 대상
- 소스 비공개로 수정/배포 가능
- 기술 지원 포함

### 3. 엔터프라이즈
- 온프레미스 설치 지원
- SLA, 전담 지원
- 커스터마이징

## 실제 경쟁력

오픈소스여도 경쟁력 유지되는 이유:

1. **호스팅 편의성**
   - 셀프호스팅은 서버 관리, 업데이트, 백업 필요
   - 월 $10-30으로 관리 부담 해소

2. **지속적 업데이트**
   - 우리가 가장 빠르게 기능 추가
   - 커뮤니티 기여도 우리에게 먼저 반영

3. **신뢰와 브랜딩**
   - 공식 서비스로서의 신뢰
   - 문서, 지원, 커뮤니티

4. **한국 시장 특화**
   - 카카오 연동, 한국 결제
   - 해외 경쟁사가 쉽게 따라오기 어려움

## 적용 계획

### Phase 1: 초기 런칭
- 위젯: MIT 라이선스로 공개
- 백엔드: 클로즈드 소스 (SaaS만 운영)

### Phase 2: 커뮤니티 성장 후
- 백엔드: AGPL로 공개
- 커머셜 라이선스 옵션 추가

### Phase 3: 성숙기
- 셀프호스팅 가이드 공개
- Docker 이미지 제공
- 엔터프라이즈 플랜 런칭
